# Meme-fluencer Frontend 프로젝트 설명서

## 1. 프로젝트 개요

Meme-fluencer는 AI를 활용하여 밈 기반 광고 영상을 자동으로 생성하는 SaaS 플랫폼입니다. 사용자가 제품 정보와 원하는 캐릭터 스타일을 입력하면, AI가 시나리오를 생성하고 영상을 자동으로 제작합니다.

이 프론트엔드는 사용자가 영상 제작을 요청하고, 각 단계별 검수를 진행하며, 완성된 영상을 관리할 수 있는 웹 애플리케이션입니다.

## 4. 라우팅 시스템

### Next.js App Router
Next.js 14의 App Router를 사용하여 파일 시스템 기반 라우팅을 구현했습니다.

### Route Groups
괄호를 사용한 Route Group으로 레이아웃을 공유하면서 URL에는 영향을 주지 않습니다:
- `(auth)` - 로그인 페이지 전용 레이아웃
- `(dashboard)` - 사용자 대시보드 레이아웃 (사이드바, 헤더 포함)

### 동적 라우트
대괄호를 사용한 동적 세그먼트:
- `/user/video/[id]` - 영상 상세 페이지 (id에 따라 동적 렌더링)

### 특수 파일
각 라우트에서 사용하는 Next.js 특수 파일:
- `layout.tsx` - 공유 레이아웃
- `page.tsx` - 페이지 컴포넌트
- `loading.tsx` - 로딩 UI (Suspense 경계)
- `error.tsx` - 에러 UI (Error Boundary)
- `not-found.tsx` - 404 페이지

## 5. 페이지별 기능

### 랜딩 페이지 (`/`)
- 서비스 소개 및 주요 기능 설명
- 로그인/회원가입 CTA 버튼
- 반응형 히어로 섹션

### 로그인 (`/login`)
- Google OAuth 소셜 로그인
- 이메일/비밀번호 로그인
- 회원가입 폼 (토글 방식)

### 메인 대시보드 (`/main`)
- 통계 카드: 총 영상 수, 총 조회수, 평균 반응률, 제작 중 영상
- 조회수 추이 차트 (Area Chart)
- 반응률 차트 (Bar Chart)
- 최근 영상 목록

### 내 영상 (`/user`)
- 영상 목록 테이블/카드 뷰
- 상태별 필터링 (대기, 생성중, 검수, 완료)
- 검색 기능
- 페이지네이션

### 영상 상세 (`/user/video/[id]`)
- 6단계 진행 상태 표시
- 캐릭터 이미지 검수 (승인/수정요청)
- 음성 샘플 검수 (승인/수정요청)
- 시나리오 검수 (승인/수정요청)
- 최종 영상 미리보기

### 영상 요청 (`/user/request`)
- 제품 정보 입력 (제품명, 카테고리, 강조 문구)
- 밈 스타일 선택 (드롭다운)
- 캐릭터 스타일 설명 (텍스트 + 작성 팁)
- 제품 이미지 업로드 (드래그 앤 드롭)
- 선택 정보 (제품 URL, 참고사항)

### 프로필 (`/profile`)
- 프로필 이미지 변경
- 닉네임 수정
- 비밀번호 변경

### 관리자 대시보드 (`/admin`)
- 전체 요청 현황 통계
- 상태별 요청 목록
- 실시간 처리 현황

### 관리자 분석 (`/admin/analytics/*`)
- 밈별 성과 분석
- 카테고리별 성과 분석
- 트렌드 분석 및 예측

## 6. 영상 제작 워크플로우

### 상태 흐름
```
대기 (pending)
    ↓
캐릭터 생성중 (character_generating)
    ↓
캐릭터 검수 (character_review) ←─── 수정 요청 시 다시 생성
    ↓ 승인
음성 생성중 (voice_generating)
    ↓
음성 검수 (voice_review) ←─── 수정 요청 시 다시 생성
    ↓ 승인
시나리오 생성중 (scenario_generating)
    ↓
시나리오 검수 (scenario_review) ←─── 수정 요청 시 다시 생성
    ↓ 승인
영상 제작중 (video_generating)
    ↓
완료 (completed)
```

### 각 단계 설명
1. **대기**: 사용자가 영상 제작을 요청한 초기 상태
2. **캐릭터 생성**: AI가 사용자 설명을 바탕으로 캐릭터 이미지 생성
3. **캐릭터 검수**: 사용자가 생성된 캐릭터 이미지를 확인하고 승인/수정요청
4. **음성 생성**: AI가 캐릭터에 맞는 TTS 음성 생성
5. **음성 검수**: 사용자가 음성 샘플을 확인하고 승인/수정요청
6. **시나리오 생성**: AI가 제품 정보와 밈을 결합하여 시나리오 작성
7. **시나리오 검수**: 사용자가 시나리오를 확인하고 승인/수정요청
8. **영상 제작**: 모든 요소를 결합하여 최종 영상 렌더링
9. **완료**: 영상 제작 완료, 다운로드 및 공유 가능

## 7. 인증 시스템

### 인증 방식
- **Google OAuth 2.0**: @react-oauth/google 라이브러리 사용
- **이메일 로그인**: 자체 인증 시스템 (백엔드 API 연동)

### 인증 상태 관리
`useAuth` 커스텀 훅을 통해 전역 인증 상태를 관리합니다:
- 로그인 상태 확인
- 사용자 정보 저장
- 로그아웃 처리
- 관리자 권한 확인

### 보호된 라우트
- `/main`, `/user/*`, `/profile` - 로그인 필요
- `/admin/*` - 로그인 + 관리자 권한(is_admin) 필요

## 8. 상태 관리

### 로컬 상태
React의 `useState`, `useEffect`를 사용한 컴포넌트 단위 상태 관리

### 폼 상태
React Hook Form을 사용한 폼 상태 관리:
- 유효성 검사
- 에러 메시지
- 제출 상태

### 테마 상태
`useTheme` 훅을 통한 다크/라이트 모드 관리:
- localStorage에 설정 저장
- 시스템 설정 감지
- CSS 변수를 통한 테마 적용

### Mock 모드
개발 환경에서 백엔드 없이 동작하는 Mock 시스템:
- localStorage를 데이터베이스로 활용
- Mock 데이터로 UI 테스트
- 환경변수(`NEXT_PUBLIC_MOCK_MODE`)로 토글
